# AGENTS.md

## Purpose
Este repositorio contiene un proyecto académico para un sitio web personal de **Lulú Tapia**. El objetivo de este archivo es orientar a agentes de IA sobre la estructura actual del proyecto, las entregas previstas y las reglas de comportamiento específicas al colaborar aquí.

## Estado actual
- `CLAUDE.md`: documento principal con contexto del proyecto, identidad visual y fases.
- `latex/`: entregable de Actividad 1 en LaTeX (`main.tex`, `main.pdf`).
- `frontend/`: carpeta vacía con `.gitkeep`; será la implementación del sitio web.
- `backend/`: carpeta vacía con `.gitkeep`; sólo se usará si hay lógica de servidor futura.

## Cómo trabajar en este repositorio
- Responder siempre en español.
- El proyecto es un trabajo académico; no agregues secciones ni contenidos fuera de lo que pide la actividad.
- Para cambios en `latex/main.tex`, compila con `pdflatex -interaction=nonstopmode main.tex` dos veces para actualizar el índice y las referencias.
- Preserva la estructura exacta de secciones en los documentos académicos; el profesor evalúa según el orden y los nombres solicitados.

## Navegación rápida
- `CLAUDE.md`: contexto del cliente, identidad visual, requisitos de usuario y fases del proyecto.
- `latex/main.tex`: entregable actual de la Actividad 1.
- `latex/main.pdf`: versión compilada del documento.

## Reglas importantes
- No inventar entregables nuevos ni crear contenido académico más allá de lo solicitado.
- No cambiar el propósito del proyecto: es un sitio web personal para una artista freelance.
- Si trabajas en la implementación del frontend, prioriza una solución limpia y simple.
- Si el repositorio no contiene código de frontend o backend aún, documenta ese estado y evita asumir stacks específicos.

## Recomendaciones para agentes
- Si se solicita implementar el sitio web, primero pregunta cuál stack o enfoque prefiere la usuaria (HTML/CSS/JS puro, framework ligero, etc.).
- Si se solicita corregir o actualizar el documento LaTeX, revisa primero `CLAUDE.md` para no romper la estructura académica.
- Usa `latex/` como la referencia de la fase completada y `frontend/` / `backend/` como las carpetas de trabajo futuro.
